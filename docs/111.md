# 每日主体搭建系统 - 阶段二（创建项目和广告）

## 概述

在已有资产化功能（创建推广链接、小程序资产、付费事件）的基础上，新增创建项目和创建广告功能，完成完整的搭建流程。

## 功能范围

### 资产化流程（已有）

- ✅ 创建推广链接
- ✅ 创建小程序资产
- ✅ 添加付费事件

### 搭建流程（新增）

- ⏸️ 创建项目
- ⏸️ 创建广告

## 用户流程变化

### 变化1：简化资产化流程

**之前**：点击"提交资产化" → 选择日期 → 拉取"待资产化"数据 → 执行资产化

**现在**：点击"提交资产化" → 直接拉取"待资产化"数据 → 执行资产化

- 移除日期选择步骤
- 直接查询飞书状态为"待资产化"的记录（不限日期）
- 其他步骤保持不变

### 变化2：新增搭建流程

**新按钮**：在"提交资产化"按钮右侧新增"提交搭建"按钮

**流程**：

1. 点击"提交搭建" → 弹出日期选择框
2. 选择日期 → 拉取飞书状态为"待搭建"且日期匹配的记录
3. 获取剧名和账户字段
4. 执行搭建流程（创建项目 → 创建广告）

## 搭建流程详解

### 步骤1：创建项目

#### 1.1 获取抖音号配置

从设置页面获取"每日配置"下的抖音号与素材序号匹配配置。

**数据结构**：

```typescript
{
  douyinAccount: "小鱼看剧",      // 抖音号名称
  douyinAccountId: "25655660267", // 抖音号ID
  materialRange: "01-04"           // 素材序号范围
}
```

**逻辑**：

- 一个抖音号 = 一个搭建批次
- 为每个抖音号创建一个项目和一个广告

#### 1.2 获取小程序资产

**接口**：

```bash
POST https://ad.oceanengine.com/event_manager/v2/api/assets/ad/list?aadvid={account_id}
```

**请求头**：

```json
{
  "platform": "ad",
  "Cookie": "{巨量后台Cookie}"
}
```

**请求体**：

```json
{
  "assets_types": [1, 3, 2, 7, 4, 5],
  "role": 1
}
```

**返回字段说明**：

| 字段                    | 说明         | 用途           |
| ----------------------- | ------------ | -------------- |
| `assets_id`             | 资产ID       | 创建项目时需要 |
| `micro_app_instance_id` | 小程序实例ID | 创建项目时需要 |

#### 1.3 创建项目

**接口**：

```bash
POST https://ad.oceanengine.com/superior/api/v2/project/create?aadvid={account_id}
```

**关键字段说明**：

| 字段                    | 值                        | 说明                       |
| ----------------------- | ------------------------- | -------------------------- |
| `name`                  | `{剧名}-小鱼-{抖音号}`    | 例如："暗香-小鱼-萍通剧坊" |
| `asset_ids`             | `[{assets_id}]`           | 从小程序资产接口获取       |
| `micro_app_instance_id` | `{micro_app_instance_id}` | 从小程序资产接口获取       |
| `budget`                | `300`                     | 固定预算                   |
| `delivery_mode`         | `3`                       | 投放模式                   |
| `pricing_type`          | `9`                       | 定价类型                   |

**固定字段**（不需要修改）：

- `product_platform_id`: "4382498222065454"
- `product_id`: "1767786539332903207"
- `landing_type`: 16
- `external_action`: 14
- `campaign_type`: 1
- `micro_promotion_type`: 4

**返回字段**：

```json
{
  "code": 0,
  "data": {
    "id": "7592808131504832575" // 项目ID，后续创建广告需要
  }
}
```

### 步骤2：创建广告

创建广告前需要准备三���数据：

1. **抖音号原始ID** (`ies_core_user_id`)
2. **小程序信息** (`app_id`, `start_page`, `params`, `link`)
3. **素材数据** (根据匹配规则过滤)

#### 2.1 获取抖音号原始ID

**接口**：

```bash
POST https://ad.oceanengine.com/superior/api/v2/ad/authorize/list?aadvid={account_id}
```

**请求头**：

```json
{
  "Cookie": "{巨量后台Cookie}"
}
```

**请求体**：

```json
{
  "page_index": 1,
  "page_size": 100,
  "uniq_id_or_short_id": "{douyinAccountId}", // 从配置中获取
  "need_limits_info": true,
  "need_limit_scenes": [4],
  "level": [1, 4, 5, 7],
  "need_auth_extra_info": true,
  "dpa_id": ""
}
```

**返回字段**：

```json
{
  "code": 0,
  "data": [
    {
      "ies_user_name": "小鱼看剧", // 抖音号名称
      "ies_core_id": "1418801121595483", // 原始ID（需要）
      "ies_id": "25655660267" // 抖音号ID
    }
  ]
}
```

**验证逻辑**：

- `ies_user_name` 必须匹配当前批次的抖音号名称
- 提取 `ies_core_id` 用于创建广告

**步骤2.2.1：查询小程序**

**接口**：

**请求体**：

```json
    "search_key": "",

```

**返回字段**：

```json
{
  "micro_app_instance_id": "7592960597470281766" // 小程序实例ID
}
```

**步骤2.2.2：获取小程序链接信息**

**接口**：

```bash
GET https://ad.oceanengine.com/superior/api/v2/ad/applet/link?aadvid={account_id}&micro_app_instance_id={micro_app_instance_id}&page=1&page_size=20&search_key=&order_type=4&app_status=1
```

**返回字段说明**：

| 字段           | 说明       | 用途                        |
| -------------- | ---------- | --------------------------- |
| `app_id`       | 小程序ID   | 创建广告时需要              |
| `start_page`   | 启动页面   | 例如："pages/theatre/index" |
| `start_params` | 启动参数   | 创建广告时需要              |
| `app_type`     | 应用类型   | 创建广告时需要              |
| `link`         | 小程序链接 | 创建广告时需要              |

**素材过滤规则**：

根据配置中的 `materialRange`（如 "01-04"）和当前剧名过滤素材。

**规则**：

1. 解析 `materialRange`：
   - "01" → 单个序号
   - "01-04" → 序号范围 01, 02, 03, 04

2. 匹配 `video_name`：
   - 必须包含当前剧名（如"玄门泰斗开局竟被当凡夫"）
   - 必须包含"-xh-{序号}.mp4"（如"-xh-01.mp4"）
   - 序号在配置的范围内

3. **示例**：
   - 配置："01-04"，剧名："玄门泰斗开局竟被当凡夫"
   - 匹配的素材：
     - ✅ "1.6-玄门泰斗开局竟被当凡夫-xh-01.mp4"
     - ✅ "1.6-玄门泰斗开局竟被当凡夫-xh-02.mp4"
     - ✅ "1.6-玄门泰斗开局竟被当凡夫-xh-03.mp4"
     - ✅ "1.6-玄门泰斗开局竟被当凡夫-xh-04.mp4"
   - 不匹配的素材：
     - ❌ "1.6-玄门泰斗开局竟被当凡夫-xh-05.mp4"（超出范围）
     - ❌ "1.6-其他剧名-xh-01.mp4"（剧名不匹配）

| `project_id`                                  | `{project_id}`                  | 上一步创建的项目ID |
| `promotion_data.native_info.ies_core_user_id` | `{ies_core_id}`                 | 抖音号原始ID       |
| `promotion_data.micro_app_info.app_id`        | `{app_id}`                      | 小程序ID           |

## 批次处理逻辑

### 剧集级别

- 一部剧集 = 多个搭建批次（每个抖音号一个批次）

### 批次级别

- 一个抖音号 = 一个批次
- 批次内执行：创建项目 → 创建广告
- 批次完成后继续下一个抖音号

### 错误处理

- 某个批次失败 → 记录错误，继续下一个批次
- 所有批次完成后显示成功/失败列表
